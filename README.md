# טופס דיווח — "הלכה עוברת"

דף אינטרנט עם הפלייר בראש העמוד וטופס דיווח בהמשכו. כל שליחה נכנסת כשורה חדשה בטבלת גוגל שיטס.

## קבצים
- `index.html` — דף הטופס (התמונה + הטופס)
- `header.jpeg` — תמונת הפלייר
- `apps-script.gs` — הקוד שמחבר לגוגל שיטס

## חיבור לגוגל שיטס (חד-פעמי, כ-3 דקות)

1. פתחי טבלת גוגל שיטס חדשה (או קיימת) שבה יתקבלו הדיווחים.
2. בתפריט: **תוספים (Extensions) ← Apps Script**.
3. מחקי את הקוד שמופיע שם, והדביקי במקומו את כל התוכן של הקובץ `apps-script.gs`. שמרי (Ctrl+S).
4. למעלה מימין: **פריסה (Deploy) ← פריסה חדשה (New deployment)**.
5. ליד "בחירת סוג" לחצי על גלגל השיניים ← **אפליקציית אינטרנט (Web app)**.
6. הגדרות:
   - **Execute as / הפעלה בתור:** אני (Me)
   - **Who has access / למי יש גישה:** **כל אחד (Anyone)**  ← חשוב!
7. לחצי **Deploy**, אשרי את ההרשאות (ייתכן מסך "Google לא אימתה" — לחצי Advanced ← Go to project), והעתיקי את הכתובת שמתקבלת (מתחילה ב-`https://script.google.com/macros/s/...`).
8. פתחי את `index.html` ובשורה שבה כתוב:
   ```
   const SCRIPT_URL = 'PASTE_YOUR_APPS_SCRIPT_URL_HERE';
   ```
   הדביקי את הכתובת במקום `PASTE_YOUR_APPS_SCRIPT_URL_HERE`.

זהו — כל דיווח ייכנס לגיליון "דיווחים" בטבלה, עם תאריך ושעה.

## פרסום הדף
אפשר להעלות את התיקייה ל-GitHub Pages (כמו מערכת הפידבק של שושי) או לכל אחסון סטטי אחר — ואז לשלוח למורות את הקישור.

## עדכון הקוד בעתיד
אם משנים את `apps-script.gs`, צריך לפרוס מחדש: **Deploy ← Manage deployments ← ✏️ ← Version: New version ← Deploy** (הכתובת נשארת אותה כתובת).
